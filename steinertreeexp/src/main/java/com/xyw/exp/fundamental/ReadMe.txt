本实验为基础实验

目标：获取可以组成 X组Category的API的斯坦纳树

步骤：
+1. 随机抽取Mashup
+2. 获取对应的APIs （随机抽取 X 组）
+3. 获取每组 APIs 对应的 一个Category（category不允许重复）
+4. 获取每组Category的所有API
+5. 对所有的API进行生长合并操作
+6. 获取第一个有效的斯坦纳树(只要节点包含需求（category）即可)并计算节点个数记为count
+7. 获取所有节点个数<=count的斯坦纳树
+8. 返回结果

斯坦纳树形成过程
1. 生长操作
1.1. 已生长过的树 不再次生长（给树做个标记 即可）
1.2. 节点自顶向上生长一层（同处于同个mashup下面的api节点），节点不可重复
1.3. 如果新生长的树 在队列中已有最短树（根节点叶子结点都相同，但是队列中的树的节点个数更少），不加入队列
1.4. 其他新生长树 放入队列
1.5. 对队列进行排序（按节点个数）
1.6. ----> 进行2操作

2. 合并操作
2.1. 不同category的树 进行合并操作
2.1. 判断是否有相同根节点，有则进行合并操作（除根节点外，有重复节点则不合并）
2.2. 合并后的树 也放在该队列中
2.3. 对队列进行排序（按节点个数）
2.4. ----> 进行1操作

结束标志： 1. 超出深度限制（暂未启用） 2. 没有新生长树 自然而止
